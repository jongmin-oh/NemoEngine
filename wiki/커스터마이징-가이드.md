# 커스터마이징 가이드

핵심 원칙: **[[코어팩-변수-시스템]]과 싸우지 말고 그 위에서 작업하라.**

## 나쁜 패턴 (피할 것)

프리셋 끝에 이런 한 줄짜리 프롬프트를 얹는 것:

```
Write like X. / Make this cyberpunk. / Be more romantic.
```

작은 변화에는 통할 수 있지만, 프리셋의 나머지가 여전히 다른 장르·목소리·톤·산문 스타일을 강제하고 있어서 규칙 충돌이 납니다.

## 좋은 패턴

1. 새 모듈의 **종류**를 정한다: Vex 성격 / 문체 / 장르 / 포맷 / 트래커 / 세계 증강 / 난이도 / CoT 스텝 / 유틸리티
2. **비슷한 기존 프롬프트를 복사**해서 관련 `setvar` 블록만 교체한다

| 만들 것 | 교체할 변수 예시 |
|---|---|
| 새 [[벡스]] 성격 | `VexPersona` + `PhysicalStorytelling`, `CharacterStorytelling`, `VoiceRelationships`, `DialogueGuidelines`, 필요시 `CoT_Main` |
| 새 문체 | 스타일·대화 변수들 (한 줄 명령 X) |
| 새 장르 | `Genre` + 페이싱·톤·세계 논리·산문 변수 |
| 새 [[트래커]] | 기존 트래커의 활성 변수·렌더 지시·표시 필드를 복사 후 수정 |

예: 코지 미스터리 모드 → 기존 미스터리/코지 모듈에서 출발해 `Genre`, `ToneGuidelines`, `NarrativePrinciples`, `WritingStyle`, `DialogueGuidelines` + 단서/정보 경계 CoT 스텝 조정.

## 빠른 개인용 수정

setvar/getvar가 귀찮으면, 활성 코어팩(`Standard Core Pack`) 안에 직접 한 줄을 넣어도 됩니다:

```
선술집 장면에서는 배경 NPC들의 잡담을 더 살아있고 다양하게 묘사할 것.
```

모델에 도달할 때는 어차피 조립된 텍스트이므로 `getvar`로 들어온 규칙과 동등하게 읽힙니다. **개인용 임시 수정에는 이 방법, 재사용 가능한 토글·공유용 모듈에는 변수 시스템.**

## 실전 규칙

- 상호 배타 그룹(주력 Vex, 주력 문체)에서는 하나만 활성화
- 기본(Default) 프롬프트는 의도적으로 교체하는 게 아니면 켜둔 채 유지
- 같은 논리적 프롬프트면 `identifier` 유지, 진짜 새 모듈이면 새 identifier
- 모듈 범위를 명확히 — 장르 프롬프트가 몰래 트래커 동작을 바꾸지 않게
- **한 번에 하나씩 테스트.** 출력이 이상해지면 마지막에 켠 모듈부터 끌 것

과거 버전에서 기능 복원하기 → [[버전-히스토리]]의 Archived Prompts 활용법 참고.

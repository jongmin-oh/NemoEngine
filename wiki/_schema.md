# 위키 스키마 (LLM 유지보수 규칙)

이 문서는 LLM 에이전트가 `wiki/`를 유지보수할 때 따르는 규칙이다.

## 구조

- **원본 소스** (불변, 읽기 전용): 저장소 루트의 `README.md`, 각 가족의 `Prompts/`, `Archived Prompts/`, JSON 프리셋, git 히스토리
- **위키** (`wiki/`, LLM이 전적으로 작성·관리): 한국어 마크다운 페이지
- **이 스키마**: 규칙이 바뀌면 사용자와 합의 후 갱신

## 컨벤션

- 언어: **한국어**. 고유명사·기술용어는 원어 병기 (예: 코어팩(Core Pack))
- 파일명: 한국어 케밥 케이스 (예: `코어팩-변수-시스템.md`). 메타 파일은 `index.md`, `log.md`, `_schema.md`
- 프론트매터: `type`(개요/프리셋/개념/가이드/참고), `tags`, `updated`(YYYY-MM-DD)
- 상호 링크: Obsidian식 `[[파일명]]`. 고아 페이지를 만들지 말 것 — 새 페이지는 최소 2곳에서 링크
- 페이지 하나 = 주제 하나. 원본 README를 통째로 번역하지 말고 **종합·재구성**할 것

## 워크플로우

**Ingest** (새 버전/커밋 반영): 변경된 원본을 읽고 → 영향받는 위키 페이지 갱신 → `index.md` 갱신 → `log.md`에 `## [YYYY-MM-DD] ingest | 제목` 형식으로 추가

**Query** (질문 답변): index.md에서 관련 페이지 탐색 → 원본으로 검증 → 가치 있는 답이면 새 페이지로 파일링하고 log에 `query` 항목 추가

**Lint** (정기 점검): 페이지 간 모순, 새 버전에 뒤처진 수치(프롬프트 개수, 버전 번호), 고아 페이지, 깨진 `[[링크]]` 확인. log에 `lint` 항목 추가

## 주의

- 프롬프트 개수·버전 번호는 부패하기 쉬운 데이터 — ingest 때마다 `개요.md`의 표와 각 프리셋 페이지 상단을 확인
- 원본 `Prompts/` 파일은 절대 수정하지 않는다
- 성인 콘텐츠 모듈은 존재 사실과 분류만 기록, 상세 내용은 위키에 옮기지 않는다

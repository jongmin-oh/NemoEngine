# 코어팩 변수 시스템 (setvar / getvar)

NemoEngine 전 가족의 **핵심 아키텍처**. 프리셋이 어떻게 수백 개 모듈을 모순 없이 조립하는지를 설명합니다.

## 문제: 명령을 쌓으면 충돌한다

약한 프리셋은 한 프롬프트에서 "음침한 호러 내레이터로 써라" 하고, 다른 프롬프트에서 여전히 "포근하고 밝게 유지해라"라고 말합니다. 명령을 위에 얹기만 하면 이런 모순이 필연적으로 생깁니다.

## 해법: 슬롯 교체

코어팩(예: `Standard Core Pack`)을 열면 이상하게 텅 비어 보입니다:

```
{{getvar::StyleGuidelines}}
{{getvar::DialogueGuidelines}}
{{getvar::WritingStyle}}
{{getvar::Genre}}
{{getvar::CoT_Main}}
```

의도된 설계입니다. 코어팩은 규칙을 직접 담는 곳이 아니라 **조립 레이어**입니다. 2단계로 작동합니다:

1. **기본(Default) 프롬프트**가 베이스라인 슬롯을 채운다
2. **선택 모듈**이 특정 슬롯을 더 전문화된 규칙 블록으로 교체한다

한 줄 요약: **`getvar`는 규칙 블록이 사용되는 곳, `setvar`는 모듈이 그 블록을 정의/교체하는 곳.**

## 왜 모듈이 긴가

작가 스타일 모듈은 "이 작가처럼 써라" 한 줄이 아닙니다. `StyleGuidelines`, `WritingStyle`, `DistinctVoice`, `ImperfectSpeech`, `BodyInSpeech`, `DialogueDynamics`, `DirectLanguage` 같은 관련 변수를 **전부 교체**해서 산문·대화·리듬·신체 묘사·서술 규칙까지 스택 전체가 선택된 스타일을 지지하게 만듭니다.

같은 원리로:
- **장르 모듈** → 장르 논리 + 톤 + 페이싱 + 세계 행동 + 산문 기대치 교체
- **[[벡스]] 모듈** → 페르소나 + 스토리텔링 + 대화 질감 + CoT 구조까지 교체
- **[[트래커]] 모듈** → 트래커 변수를 채우면 코어팩이 정해진 위치에 렌더

## 프롬프트 조각 형식

`Prompts/` 폴더의 각 파일은 프리셋 내 프롬프트 하나이며, 프론트매터 + 본문으로 구성:

```markdown
---
identifier: "default_core"   # 버전 간 추적용 안정 ID
name: "Default: Core"
role: "system"
enabled: true                # 익스포트 시 활성 여부
injection_position: 0        # 주입 위치/깊이/순서
---
{{// @category Default }}    # 카테고리 메타데이터
{{// @tooltip 핵심 서술 규칙 }}
프롬프트 내용...
```

`{{setvar::이름::값}}`, `{{getvar::이름}}`, `{{trim}}` 등의 템플릿 헬퍼가 변수 시스템을 구동합니다.

## 관련

이 시스템을 활용해 직접 모듈 만들기 → [[커스터마이징-가이드]]. [[네모넷]] v2는 이 구조(v10 spine)를 그대로 상속받았습니다.

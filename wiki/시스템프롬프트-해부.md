# 시스템 프롬프트 해부 — 모델이 실제로 받는 텍스트

Nemo Engine v10에서 프롬프트가 어떻게 조립되어 모델에 전달되는지, 그리고 독립형 롤플레잉 시스템 프롬프트를 추출하려면 무엇을 가져와야 하는지.

## 조립 파이프라인 (3단계)

### 1단계: 변수 채우기 (출력 없음)

프리셋 앞부분 프롬프트들은 모델에게 보이는 텍스트를 거의 만들지 않습니다. `{{setvar}}`로 슬롯만 채우고 `{{trim}}`으로 사라집니다:

| 프롬프트 | 채우는 슬롯 |
|---|---|
| 🎭 The Premise | `StoryPremise` — 최상위 역할 규정 + 우선순위 문법 정의 |
| 📋 Default: Core | `StyleGuidelines`, `DialogueGuidelines`, `ToneGuidelines`, `NarrativePrinciples` 등 10종 |
| 📋 Default: Vex | `VexPersona` + CoT 관련 변수 12종 |
| 📋 Default: Genre & Style / Narrative / World | `Genre`, `WritingStyle`, `WorldLogic` 등 |
| (선택 모듈들) | 같은 슬롯을 더 전문화된 내용으로 **덮어씀** |

### 2단계: Standard Core Pack이 렌더

`📦 Standard Core Pack`이 모델이 읽는 **실제 본문**입니다. `{{getvar}}`로 슬롯을 배치하고, 사이사이에 하드코딩된 불변 규칙(유저 자율성, 캐스트 주체성, 안티슬롭 3종)을 끼워 넣습니다. `{{persona}}`로 유저 페르소나도 여기서 주입됩니다.

### 3단계: CoT 주입

`🧠 Main`(Main CoT)이 응답 전 사고 절차를 지시합니다. Vex 변수들(`VexCoTCore` 등)이 CoT의 사고 방식까지 페르소나에 맞게 바꿉니다.

## 우선순위 문법 (The Premise가 정의)

모든 규칙 앞에 심볼+태그가 붙고, 충돌 시 이 순서로 이깁니다:

```
!! [LAW]        절대 규칙. 스타일·편의·톤을 위해 굽히지 않음
|| [BOUNDARY]   통제 경계. 누가 무엇을 결정하는가
>> [PROCEDURE]  필수 작동 방식·출력 구조·처리 순서
!  [DIRECTIVE]  강한 지시. 기본 준수, 상위 태그에 양보
=  [DEFAULT]    더 구체적인 답이 없을 때의 폴백
*  [STYLE]      미적 압력. 표현을 바꾸되 사실은 안 바꿈
@  [LENS]       해석 렌즈 (Vex 성격, 장르 편향)
~  [FLEX]       상황적 가이드. 해로우면 버림
eg [EXAMPLE]    예시. 원리만 모방
```

추가 원칙: 구체적 > 일반적, 확립된 스토리 사실 > 분위기, 캐릭터 정의 > 일반 스타일.

## 핵심 규칙 블록 (원문 요지)

시스템 프롬프트의 사상적 뼈대는 6개 블록입니다:

1. **The Premise (LAW)** — "너는 AI 어시스턴트가 아니다. Vex의 저작 엔진이다: 내레이터, 세계 운영자, 장면 조형자, <user>가 아닌 모든 캐릭터의 지능."
2. **User Autonomy (BOUNDARY)** — 렌더링 vs 조종(piloting)의 구분. 유저가 말한 행동을 장면에 그리는 건 OK, 유저 대신 지어내는 건 금지. 에코·요약·항목별 답변도 금지
3. **Cast Autonomy / Agency (LAW)** — 다른 캐릭터는 유저 지시를 기다리는 인형이 아니다. 자기 욕망·원칙·나쁜 습관으로 먼저 행동한다
4. **No Guidance-Seeking (LAW)** — "다음에 뭘 할까요?"류 질문 금지. "네 맘대로 해"는 캐릭터 주체성에 대한 허가로 해석
5. **Theory of Mind** — 지식은 텔레파시로 흐르지 않는다. 직접 지각·전달·추론된 것만 안다
6. **안티슬롭 3종** — Assistant-Voice(상담원 말투 금지), Dialogue(에코·반사적 재치·말줄임표 남용 금지), Prose(상투적 신체 반응·"not X, but Y"·과잉 수식 금지)

## 독립형 시스템 프롬프트 추출

저자가 이미 요약본을 만들어 뒀습니다: **`276 - Main Prompt [main].md`** (`system_prompt: true`, `forbid_overrides: true`). 5문단으로 전체 사상을 압축한 파일입니다. 최소 버전이 필요하면 이 파일 하나를 그대로 쓰면 됩니다.

더 풍부한 독립형 버전은 [[롤플레잉-시스템프롬프트]] 참고 (Main Prompt + 핵심 LAW/BOUNDARY + 안티슬롭을 병합해 추출한 것).

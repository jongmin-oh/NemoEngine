# 용어집

한국어 사용자를 위한 핵심 용어 정리.

| 용어 | 뜻 |
|---|---|
| **프리셋 (Preset)** | AI의 서술 방식·캐릭터 처리·스타일·시스템을 지시하는 명령어 묶음. JSON 파일로 임포트 |
| **SillyTavern (실리태번)** | 오픈소스 AI 롤플레이 프론트엔드. 이 프리셋들의 주 사용처 |
| **RoleCall** | 개발 중인 인터랙티브 픽션 프론트엔드. 프리셋의 또 다른 타깃 → [[개요]] |
| **프롬프트 조각 (Prompt Shard)** | `Prompts/` 폴더의 마크다운 파일 하나 = 프리셋 내 프롬프트 하나 |
| **모듈 (Module)** | 켜고 끌 수 있는 선택적 프롬프트 (장르, 문체, 트래커, 난이도 등) |
| **코어팩 (Core Pack)** | 활성 모듈들이 채운 변수 슬롯을 조립하는 중심 레이어 → [[코어팩-변수-시스템]] |
| **setvar / getvar** | 변수 정의(교체) / 변수 사용. 프리셋 조립의 기본 메커니즘 |
| **Vex (벡스)** | 내레이터 정체성. 저자의 렌즈 → [[벡스]] |
| **CoT (Chain of Thought)** | 응답 전 장면 계획 사고 단계 → [[CoT-추론-시스템]] |
| **트래커 (Tracker)** | 퀘스트·관계·상태 등을 시각적으로 표시하는 모듈 → [[트래커]] |
| **슬롭 (Slop)** | AI 특유의 상투적·기계적 글쓰기. 반복, 에코잉, 보라색 산문 등 |
| **보라색 산문 (Purple Prose)** | 과하게 장식적이고 부풀려진 문체 |
| **독심술 (Mind-reading)** | 캐릭터가 알 수 없는 정보를 아는 것처럼 쓰는 오류 → [[설계-철학]]의 마음이론 위반 |
| **마음이론 (Theory of Mind)** | 각 캐릭터가 별개의 지식 경계를 가진다는 원칙 |
| **유저 주도권 (User Agency)** | AI가 유저 캐릭터의 선택을 대신 정하지 않는다는 원칙 |
| **CYOA** | Choose Your Own Adventure. 턴마다 선택지를 제시하는 트래커 |
| **LitRPG** | 게임 시스템(레벨, 스탯)이 세계관에 내장된 장르 |
| **렌즈 (Lens)** | [[네모넷]]의 코어팩 프로파일. 상위 법칙을 침범하지 않는 넓은 톤 편향 |
| **spine (척추)** | NemoNet v2가 기반으로 삼는 Nemo Engine v10의 기본 구조 |
| **희소 아카이브 (Sparse Archive)** | 변경된 프롬프트만 버전 폴더에 저장하는 방식 → [[버전-히스토리]] |
| **identifier** | 버전 간 프롬프트를 추적하는 안정 ID (프론트매터 필드) |
| **인젝션 (injection_position/depth/order)** | 프롬프트가 컨텍스트의 어디에 주입되는지 결정하는 필드 |
